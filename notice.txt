오늘 할 일
1. 헤더and사이드바 기존 HTML 소스에서 views/partial로 분리 (조연오)
	1-1. 헤더에는 키를 포함한 API 호출문을 포함할 것임(매우 간단한 작업, 테스트만 거치는 수준임) (조규형)
	1-2. CSS 속성 관련하여 분류작업 (조연오)
        public/css/
        ├── base/
        │   ├── reset.css          # 브라우저 기본 스타일 초기화
        │   └── typography.css     # 폰트, 글자 크기 등 공통 설정
        ├── components/            # 재사용 가능한 컴포넌트별 CSS
        │   ├── navbar.css         # 네비게이션 바
        │   ├── button.css         # 공통 버튼 스타일
        │   └── card.css           # 매치 목록의 카드 아이템
        ├── layouts/               # 페이지 레이아웃 관련
        │   ├── main-layout.css    # 메인 페이지 레이아웃
        │   └── auth-layout.css    # 로그인/회원가입 레이아웃
        └── pages/                 # 특정 페이지 전용 CSS
            ├── match-detail.css   # 매치 상세 정보 페이지 전용
            └── ranking.css        # 랭킹 페이지 전용

        위와 같은 폴더 구조 구축 후 
        public/css/style.css에서 

        @import "./base/reset.css";
        @import "./components/navbar.css";
        @import "./pages/match-detail.css";
        
        롤 통해 효율적으로 css구문 관리 가능
        이 작업 수행 시 반드시 제미나이 도움 받아서 작업하는 걸 추천(클로드로 하기에는 너무 저차원 작업이고 토큰 낭비)
2. 로그인 & 회원가입 기능 (유승환, 이윤재)
    2-1. 조연오와 협업하여 실질적인 로그인 기능 구현(모양까지 거의 매치시키라는 뜻임)
    2-2. 이 작업은 존나 중요하니 코딩 한 턴하고 제미나이한테 코드 짤 때 의도 어필하면서 검사 받을 것
    2-3. 이 작업은 진짜 진짜 존나 존나 중요하니 오래걸려도 상관없음(이번 주에만 하면 됨)
    2-4. 보안이나 예외처리 관련 부분을 신경 많이 썼으면 좋겠음
    2-5. 시간이 된다면 네이버나 구글로 로그인까지 구현(이번 주에 안 되더라도 다음 기능 구현하고 틈틈히 넣을 거긴 함)

3. 마이페이지 기능 (신승민, 이은수)
    3-1. 네이버, 구글, 각종 커뮤니티나 쇼핑몰 사이트의 마이페이지를 고루 탐색하여 어떤 구성이 더 필요할지,
        필요 없을지 잘(제발) 생각하여 요소 가감할 것
    3-2. 3-1에 의해 요소 가감 여부를 결정했다면 실행 전 반드시 제미나이에게 검사받고 변경 예정사항을 팀장에게 보고할 것
    3-3. 3-2의 실행 결정 시 데이터베이스 모델 스키마나 타 기능의 구성 디테일 변경이 필요하다면 즉각 보고할 것

4. 데이터베이스 모델 스키마/메소드, API 응용 메소드, 향후 계획 (조규형)
    4-1. 1,2,3을 구현한 후의 계획은 매칭 생성 및 검색, 고정 크루 생성 및 검색임
        따라서 해당 기능 구현에 지연이 없도록 관련 메소드 1차적으로 구현할 것임
    4-2. 1,2,3을 구현하며 당장 필요하거나 4-1의 계획에서 필요할 것으로 예상되는 응용 메소드(무슨무슨 정보 넣어서
        무슨무슨 값 반환받는 메소드가 필요하다 싶은거)가 있다면 반드시 언급해줘야 함
    4-3. 4-1의 계획 이후는 고정 크루 채팅, 홍보글 작성, 팔로우OR친구추가
        그 이후는 명예OR매너좋아요 시스템, 스포츠 별 게시판(거래 포함 시킬 시간 있으면 함)

5. 공통 참고 사항
    5-1. 모르겠거나, 좆됐거나, 이의 있으면 바로 말해야 함
    5-2. 코드 이해 안 가는데 막 가져다 쓰지 말고 왜 이렇게 했을까하는 의구심과 책임감을 가지고 최소한 메소드 간의
        상호작용과 데이터 흐름은 알고 코드 짜야 함
    5-3. 5-2에서 제미나이 봐도 이해 안가면 도와달라 해야 함
    5-4. 기능 구현 끝났거나 할 거 없으면 적극적으로 어필
    5-5. 에러 메시지도 나와있는 줄까지는 쫒아가보고 모르겠으면 AI한테 물어볼 것
    5-6. 천천히 구현해도 되니까 제발 정확하고 이해된 코드만 써주세요
